Module Three Project Log - Team Development: QA and Testing Plan: 
 
 Team Green – QA & Testing Plan 
 
 
 Collaboration and Communication 
Our team will continue using the communication methods established in Module Two: 
 
- **Primary Communication:** Discord   
- **Secondary Communication:** Email   
- **Project Tracking:** GitHub Issues and Project Board   
- **Meetings:** Weekly meetings and midweek check-ins   
 
This ensures consistent collaboration and efficient updates to the QA/testing plan.   
 
--- 
 
 Testing Process Overview 
Our testing plan is integrated into the development lifecycle to ensure a stable and playable top-down stealth/action game. 
 
Testing will occur in three major stages: 
- **Play Test (Alpha / Preproduction)** 
- **Demo Test (Beta / Pre-release)** 
- **Code Release Testing (Final Verification)** 
 
Testing will focus on: 
- Core gameplay systems (movement, collisions, AI) 
- Level functionality and progression 
- Stability and performance 
- User experience (UI/UX) 
 
--- 
 
Testing Schedule 
 
Play Test (Preproduction – Alpha Stage, Week 5) 
**Purpose:** Validate core gameplay systems early   
 
**Process:** 
- Internal testing by team members   
- Test player movement, collisions, and basic enemy behavior   
- Rapid feedback through Discord   
- Bugs documented in GitHub Issues   
 
**Expected Outcome:** 
- Functional playable level   
- Identification of major bugs and gameplay issues   
 
--- 
 
Demo Test (Beta Stage, Week 8) 
**Purpose:** Ensure game is ready for demonstration   
 
**Process:** 
- Full playthrough by team members   
- Collect feedback from testers   
- Test completed systems: 
  - Level design and flow 
  - Power-ups and enemies 
  - UI/UX improvements 
- Evaluate stability   
 
**Expected Outcome:** 
- Fully playable and stable game   
- Minimal bugs remaining   
 
--- 
 
 Code Release Testing 
**Purpose:** Verify final build meets quality standards   
 
**Process:** 
- Perform regression testing   
- Confirm all reported bugs are resolved   
- Validate all game features   
 
**Expected Outcome:** 
- Stable release-ready build   
- No critical or game-breaking issues   
 
--- 
 
 Testing Checklist (Pass/Fail) 
 
Core Gameplay 
- Player movement functions correctly   
- Player spawns at correct starting location 
- Camera follow the player properly 
- Player can aim and attack as intended 
- Collision system works properly   
-  Health system operates correctly   
 
 Enemies 
- Moving enemies patrol correctly   
- Stationary enemies behave correctly   
- Enemy damage interaction works   
- Enemy movement supports level design 
 
 Level Design 
- Player can navigate all rooms   
- Start and end points function correctly   
- Obstacles affect movement as intended   
 
UI 
- Health bar displays correctly 
- Ammo indicator or gameplay information displays correctly 
- Crosshair or aiming indicator appears correctly 
- Minimap or map indicator is readable if implemented 
- UI updates when player health, ammo, or power-up status changes 
 
 
 
Game Features 
- Power-ups activate correctly   
- UI displays health and score accurately   
- Visuals and sound are consistent   
- At least 3 pickup types are implemented 
 
Performance 
- Game runs without crashing   
- No major lag or performance issues   
 
UI 
- Build can be played from start to finish 
- Required elements are included 
- Game is functional with placeholder assets 
- Major bugs are identified and reported 
- Game is ready for Alpha testing and review 
 
--- 
 
 Updating the Test Plan 
The QA plan will evolve as the game develops: 
 
- Updates will be discussed during weekly meetings   
- Changes will be applied to the README file   
- GitHub commits will track all modifications   
- New test cases will be added when features are introduced   
 
--- 
 
## Bug Reporting Process 
All bugs will be reported using **GitHub Issues**. 
 
Each report will include: 
- Bug title   
- Steps to reproduce   
- Expected vs actual result   
- Severity level (Low / Medium / High)   
- Screenshots (if applicable)   
 
Critical issues will be communicated immediately through Discord.   
 
--- 
 
Bug Tracking Over Time 
Bug tracking will be managed using GitHub workflow: 
 
- **Open → In Progress → Resolved → Closed**   
- Bugs assigned to team members   
- Issues categorized (UI, gameplay, performance)   
- Weekly review of outstanding bugs   
 
This ensures accountability, visibility, and continuous quality improvement.
